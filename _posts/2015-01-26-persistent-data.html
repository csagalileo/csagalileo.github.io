This lessons focuses on data persistence. It enhances the Android Mash game by adding a TinyDb component, which enables the app to remember a person's best score. 

Objectives: In this lesson you will learn to

  • create an app that saves the player’s high score between sessions;
  • learn about the concept of persistent data;
  • learn how to access App Inventor’s TinyDB component.

Open App Inventor with the TinyDb Template in a separate tab and follow along with the tutorial. Once the project opens use Save As to rename your project AndroidMashTinyDb. Note: If the blocks don’t appear well in the Blocks Editor, right-click on the background and use the Arrange Vertically option.

Mini-Enhancement: Try Storing a Value in a TinyDB

Whenever the player achieves a top score, store the current date in the TinyDb. 

Hints: You don’t need a second TinyDB for this task. You just need a second tag, e.g., “date”. To get the current date and time, you can use the Clock component, which is named AndroidTimer. The following blocks will return a text formatted something like this: MM/DD/YY HH/MM/SS AM. 

Reflection

Create a page named Reflection 27 under the Reflections category of your portfolio and answer the following questions.

  • What does it mean to say that data is persistent?
  • Describe the differences between data stored in a global variable and in a TinyDb.
