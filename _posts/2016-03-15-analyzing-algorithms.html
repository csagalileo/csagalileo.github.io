--- layout: post title: Analyzing Algorithms categories: csp2016 --- Searching and Sorting Experiments. In this lesson we are going to use Python to experiment with and analyze the algorithms we have been studying. You will be running two programs, one to test the search algorithms and one to test the sorting algorithms.

This activity will resemble that of a scientific investigation. You'll run the programs repeatedly on different lists of data, record the running times of the algorithms, tabulate and graph your data, and then analyze the results. Can you figure out from the results, which algorithm is which?

Another way to look at this activity is as quality assurance (QA). Many careers in the computing field start with assignments in QA. This is where you help software developers test and debug their programs.

Objectives: In this lesson you will learn to :

  • conduct an empirical (experimental) investigation of basic search and sort algorithms;
  • think analytically about basic search and sort algorithms;
  • deepen your understanding of basic search and sort algorithms.

Search Experiment

Empirical Search Analysis. In this activity you are going to use a Python program to experiment with and analyze the binary and sequential search algorithms. Search program is available here: search.py

  1. You will be performing a worst case analysis of the algorithms. Whenever you enter a number for the list size, the program will search for a number that isnot in the list.
  2. Test each search algorithm on lists of size 100, 200, ..., 100000 numbers. NOTE:Because these algorithms involve loops, you may have to wait a while for your program to finish.
  3. Create a table and write the search time in milliseconds (ms) for each test case. You should have a total of 20 test cases, 10 for each algorithm.
  4. Graph your results. You may want to use Google Sheets to generate your graph and copy the graph to your assignment doc.
  5. Analyze your results to determine which algorithm is which. Which is the binary and which is the sequential search? Provide a clear description, referring to your graph and your tabulated data, to explain how you arrived at your conclusion.

Sort Experiment

Empirical Sort Analysis. In this activity you are going to use a Python program to experiment with and analyze the bubble, merge, and bucket sort algorithms. Sorting program is available here: sorting.py

  1. Test each sort algorithm on lists of size 10, 20, ..., 10000 numbers. NOTE: Because these algorithms involve loops, you may have to wait a while for your program to finish.
  2. Create a table and write the sorting time in milliseconds (ms) for each test case. You should have a total of 30 test cases, 10 for each algorithm.
  3. Graph your results. You may want to use Google Sheets to generate your graph and copy the graph to your assignment doc.
  4. Analyze your results to determine which algorithm is which. Which is the bubble, and which is the merge, and which is the bucket sort. Provide a clear description, referring to your graph and your tabulated data, to explain how you arrived at your conclusion.
