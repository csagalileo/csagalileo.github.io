This lesson focuses on shareable data. It further enables the Android Mash game by using a TinyWebDb component to record and share auniversal top score among the game's users. 

Objectives: In this lesson you will learn to:

  • create an app that shares the highest score among several players;
  • understand the concept of sharing Web data;
  • grasp the difference between synchronous andasynchronous operations;
  • access App Inventor’s simple TinyWebDB component.

Open App Inventor with the TinyWebDb Template in a separate tab and follow along with the following tutorial. Once the project opens use Save As to rename your project AndroidMashTinyWebDb. Note: If the blocks don’t appear well in the Blocks Editor, right-click on the background and use the Arrange Vertically option.

  • Video tutorial

Mini-Enhancement: Try Storing a Value in the TinyWebDB

Use TinyWebDB to store and retrieve the name of the player with the top universal score. 

Hint: You don’t need a second TinyWebDB for this project. You just need a second tag, e.g., “AndroidMashTopName”. Don’t forget to display the top player’s name and score in the app.
